JVM内存模型

共享堆

对象、对象成员(不管是不是原生类型) 与类定义、静态变量都在堆上。

堆中的所有对象，可以被所有线程访问，只要他们能拿到对象的引用地址

Xmx : 原则上不超过物理内存的70%，因为其他还要占用，否则jvm看还没达到一直试图分配内存，但是物理机没有了，则OOM。默认分配是四分之一

Xms :

Xss

线程栈

原生数据类型的局部变量就保存在线程栈上。

栈的深度初始化的时候就已经确定，只存对象的引用，需要的时候去堆里取数据，进行计算（可以看作小的CPU）。具体的对象内容都是保存在堆中。

每启动一个线程，jvm就会在栈空间分配一个线程栈，比如Xss 1M

jstat / jstack / jmap 查看jvm参数配置

GC

老年代：经过几次GC没有干掉的 / 或者一个大对象
